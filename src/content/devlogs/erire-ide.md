The Erire IDE is being shaped around one principle: development should stay direct.

Writing code, organizing files, running projects, and building applications should happen inside one clear workflow instead of being scattered across too many tools.

The goal is not only to edit code, but to make the whole environment feel focused and usable from the start.

Current work is centered on responsiveness, project handling, and a smoother path from syntax to a working application.
